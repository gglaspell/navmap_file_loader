# navmap_file_loader

ROS 2 node that loads a navmap file generated by [bag_to_navmap](https://github.com/gglaspell/bag_to_navmap) for use in EasyNavigation. 

## Terminal Checklist: `bag_to_navmap` → `navmap_file_loader` → EasyNavigation

Everything below assumes your ROS 2 workspace is at `~/ros2_ws` and your `.navmap` file is at `/data/output/tb3.navmap`. Adjust both paths to match your setup.

***

### Phase 1 — Generate the `.navmap` file

**Terminal 1**

```bash
# Run your offline pipeline from bag_to_navmap
cd ~/ros2_ws/src/bag_to_navmap

docker run --rm \
  -v /path/to/your/bag:/data/input \
  -v /path/to/output:/data/output \
  bag_to_navmap \
  /data/input/your_bag /data/output \
  --pc_topic /points \
  --odom_topic /odom \
  --enable_loop_closure \
  --level_floor

# Verify the file exists and is non-zero
ls -lh /data/output/*.navmap
```


***

### Phase 2 — Build the workspace

**Terminal 1**

```bash
cd ~/ros2_ws

# Place navmap_file_loader under src/ if not already there
# ls src/navmap_file_loader/

colcon build --packages-select navmap_file_loader --symlink-install

# Confirm the plugin library was installed
ls install/navmap_file_loader/lib/libnavmap_file_loader.so

# Confirm the plugin XML was installed and registered
cat install/navmap_file_loader/share/navmap_file_loader/navmap_file_loader_plugins.xml

source install/setup.bash
```


***

### Phase 3 — Configure the map path

**Terminal 1**

```bash
# Edit the YAML to point at your actual .navmap file
nano ~/ros2_ws/src/navmap_file_loader/config/navmap_file_loader.yaml
```

Set `map_file` to your real path:

```yaml
maps_manager_node:
  ros__parameters:
    use_sim_time: false
    map_types: ["navmap"]
    navmap:
      plugin: "easynav::NavmapFileLoader"
      map_file: "/data/output/tb3.navmap"
      frame_id: "map"
```

```bash
# Rebuild to push the updated YAML into install/
colcon build --packages-select navmap_file_loader --symlink-install
source install/setup.bash
```

> **Note:** If you used `--symlink-install`, the YAML edit is live immediately and you do not need to rebuild.

***

### Phase 4 — Launch the maps manager node

**Terminal 1**

```bash
source ~/ros2_ws/install/setup.bash

ros2 launch navmap_file_loader navmap_file_loader.launch.py
```

Expected output:

```
[maps_manager_node]: Loading MapsManagerBase navmap [easynav::NavmapFileLoader]
[maps_manager_node]: Loaded MapsManagerBase navmap [easynav::NavmapFileLoader]
[maps_manager_node]: [NavmapFileLoader] initialized. map_file='/data/output/tb3.navmap', frame_id='map', topic='navmap/map'
```


***

### Phase 5 — Lifecycle: configure and activate

`MapsManagerNode` is a lifecycle node and must be configured and activated before your plugin's `on_initialize()` and `update()` are called.

**Terminal 2**

```bash
source ~/ros2_ws/install/setup.bash

# Confirm the node is visible and in Unconfigured state
ros2 lifecycle list /maps_manager_node

# Configure (calls on_configure -> loads your plugin -> calls on_initialize)
ros2 lifecycle set /maps_manager_node configure
```

Expected output in Terminal 1:

```
[NavmapFileLoader] initialized. map_file='/data/output/tb3.navmap', frame_id='map', topic='navmap/map'
```

```bash
# Activate (starts the update cycle -> plugin update() begins running)
ros2 lifecycle set /maps_manager_node activate
```

Expected output in Terminal 1 once a subscriber connects:

```
[NavmapFileLoader] NavMap loaded from '/data/output/tb3.navmap', waiting for subscriber...
```


***

### Phase 6 — Verify the topic

**Terminal 2**

```bash
# Confirm the topic exists
ros2 topic list | grep navmap

# Expected:
# /maps_manager_node/navmap/map

# Check message rate (should fire once then go silent)
ros2 topic hz /maps_manager_node/navmap/map

# Inspect the message content
ros2 topic echo /maps_manager_node/navmap/map --once

# Check publisher QoS matches plain depth=1 (NOT transient_local)
ros2 topic info /maps_manager_node/navmap/map --verbose
```

Expected `topic info` output:

```
Publisher count: 1
  ...
  QoS: Reliability: RELIABLE, Durability: VOLATILE, ...
```

> **If the topic does not appear:** the lifecycle node was not activated. Re-run Phase 5.

> **If the topic appears but no message fires:** no subscriber has connected yet. The plugin waits for `get_subscription_count() > 0` before publishing.  Connect a subscriber (e.g. `ros2 topic echo`) and the message will fire immediately.

***

### Phase 7 — Connect EasyNavigation

**Terminal 3**

```bash
source ~/ros2_ws/install/setup.bash

# Launch whatever EasyNavigation bringup your robot uses.
# The planner and localizer will subscribe to /maps_manager_node/navmap/map.
ros2 launch easynav easynav.launch.py
```

```bash
# In Terminal 2 — confirm the navmap was delivered to EasyNav
ros2 topic echo /maps_manager_node/navmap/map --once
```

Expected output in Terminal 1 once EasyNav subscribes:

```
[NavmapFileLoader] Published NavMap on 'navmap/map'
```

> **If EasyNav expects a bare `/navmap` topic instead**, uncomment the remapping in `launch/navmap_file_loader.launch.py`:
> ```python > remappings=[ >     ("maps_manager_node/navmap/map", "navmap"), > ], > ```
> Then rebuild and relaunch.

***

### Quick-reference command summary

```bash
# Phase 1 — generate map
docker run ... bag_to_navmap /data/input/bag /data/output --pc_topic /points ...

# Phase 2 — build
colcon build --packages-select navmap_file_loader --symlink-install && source install/setup.bash

# Phase 3 — edit YAML map_file path (if not using --symlink-install, rebuild after)

# Phase 4 — launch
ros2 launch navmap_file_loader navmap_file_loader.launch.py

# Phase 5 — lifecycle
ros2 lifecycle set /maps_manager_node configure
ros2 lifecycle set /maps_manager_node activate

# Phase 6 — verify
ros2 topic list | grep navmap
ros2 topic info /maps_manager_node/navmap/map --verbose
ros2 topic echo /maps_manager_node/navmap/map --once

# Phase 7 — launch EasyNav
ros2 launch easynav easynav.launch.py
```
